# The-Algorithmic-Labyrinth
AI-powered dungeon crawler where a self-improving Boss NPC adapts DSA, logic, and math challenges to each player using Cognee's memory lifecycle + RL.

Dungeon of Recall is a single-player dungeon-crawler built for the Cognee "Hangover Part AI" hackathon. Each floor gates progress behind a coding (DSA), logic, or math challenge that scales in difficulty as you descend — easy puzzle rooms, medium mini-bosses, and a hard final boss, plus a hidden side-dungeon of math/logic riddles.
The twist: the Boss/NPC AI actually remembers you. Using Cognee's full memory lifecycle — remember(), recall(), improve()/memify, and forget() — combined with a lightweight Sarsa(λ)-style reinforcement learning layer over the knowledge graph, the dungeon master tracks your patterns (weak topics, time-under-pressure, retry behavior) and adapts question selection and boss strategy across sessions. No more facing the same static difficulty curve twice.

**Engineering note (reinforcement mechanism):** outcome reinforcement is implemented primarily via **graph-enriched feedback episodes** — each fight outcome is written back through `remember()` as a first-class "Boss profile correction" episode that changes what the Boss's next `recall()` sees — with cognee's session-scoped QA feedback (`session.add_feedback` + `improve(session_ids=...)` feedback weights) layered on top opportunistically. We chose this at hackathon time after finding that on demo-sized graphs, session feedback weights only re-rank an already-fully-retrieved context (top_k ≥ graph size), and cognee 1.2.2 has a session/permission interaction that hides QA ids from explicit-`query_type` recalls. Both channels run; the episode channel is the load-bearing one. Verified end-to-end by `scripts/smoke_memory.py` (all four verbs PASS live when the LLM endpoint is responsive).

**Known limitation (LLM latency):** `smoke_memory.py` occasionally fails `remember`/`improve` with *timeout-only* degradations (20–25s outage guards tripping) due to intermittent latency on the aimlapi.com endpoint — not a logic error. The extraction-quality checks (`recall`, `recall/staleness`, `forget`) pass every run and profile JSON is never malformed. See `backend/memory/API_NOTES.md` → "Known flakiness: aimlapi LLM latency".
